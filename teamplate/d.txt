1. Filosofia do board

O board precisa ser guiado por estes princípios:

1. Escaneabilidade acima de decoração

O usuário precisa entender a coluna e o card em milissegundos.

2. Densidade útil

Tem que ter informação suficiente para operar sem abrir tudo, mas sem ficar pesado.

3. Movimento leve

Hover, drag, open drawer e quick actions devem parecer naturais.

4. Consistência entre kanban e tabela

Os dois modos precisam compartilhar:

status
chips
semântica de cores
estrutura de leitura
5. O drawer é o “centro da tarefa”

O card mostra o necessário.
O drawer resolve o restante.

"use client";

import { useState } from "react";
import { AppShell } from "@/components/layout/app-shell";
import { PageHeader } from "@/components/layout/page-header";
import { BoardToolbar } from "@/components/board/board-toolbar";

export default function BoardPage() {
  const [view, setView] = useState<"kanban" | "table">("kanban");

  return (
    <AppShell>
      <div className="space-y-6">
        <PageHeader
          title="Board"
          description="Gerencie tarefas com um fluxo mais confortável, visual e escaneável."
          actions={

        <BoardToolbar view={view} onChangeView={setView} />

        {view === "kanban" ? <BoardKanbanView /> : <BoardTableView />}
